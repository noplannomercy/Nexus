## 1. 과제 현황

GCore 레거시 코드베이스(2018년 이후 문서화 단절)의 업무 지식이 **특정 개인의 경험에 종속**되어 있으며, 이는 온보딩·유지보수·정책 대응의 지속적 병목으로 작용하고 있는 것으로 이해하고 있다. 본 과제는 이를 **Q&A 챗봇 기반 지식베이스로 구조화**하는 것을 목표로 하며, 파일럿 완료 시점은 **2026년 11월**로 합의되어 있다.

---

## 2. 제안하는 방향

```
                    ┌─────────────────────────────────┐
                    │         Forge (전처리 엔진)       │
[Bitbucket] ──→     │  PL/SQL  → Reverse Doc Service  │ ──→ [인간 검증] ──→ [LightRAG] ──→ [챗봇 질의]
                    │  문서    → Document Parser       │                         │
                    │  코드    → Graphify ─────────────┼──→ [Apache AGE]         │
                    └─────────────────────────────────┘         │               │
                                                                 └───────────────┘
                                                              질의 시 context 결합
                                                                         │
                                                              [PostgreSQL 단일 저장소]
```

**핵심 원칙 3가지**:

| 원칙 | 의미 |
|---|---|
| **검증 가능성 우선** | 모든 답변에 소스 링크 첨부. 근거 부족 시 "확인 불가" 명시. 할루시네이션 원천 차단 |
| **인간 승인 루프 내장** | 자동 생성 문서는 승인 후에만 지식베이스 편입. 금융권 감사 대응 가능 |
| **단일 데이터 레이어** | PostgreSQL 하나로 전체 지식 자산 관리. 운영 단순성 + 백업·복구 용이 |

## 3. HCA Code2Rule과의 관계

| 구분 | HCS (본 과제) | HCA Code2Rule |
|---|---|---|
| 목적 | 온보딩·업무 질의 지원 | 비즈니스 룰 자동 추출 |
| 산출물 | 챗봇 + 검증된 문서 | 룰 명세서 |
| 착수 | 진행 중 | 2026년 5월 |

→ **상호 보완 관계**이며 중복 과제가 아니다. HCA 양원모 과장과 정기 동기화 채널을 유지한다.

## 4. 확정이 필요한 사항 (HCS 측 결정)

아키텍처 최종 확정 및 다음 단계 착수를 위해, 아래 3개 항목의 **HCS 측 정책 결정이 선행되어야** 한다.

| # | 항목 | 영향 | 우선순위 |
|---|---|---|---|
| 1 | **소스코드 외부 API 반출 정책** | 전체 아키의 분기점 | **Critical** |
| 2 | AWS Bedrock 계약 상태 | LLM 추론 경로 결정 | High |
| 3 | 보안·네트워크 정책 범위 | 배포 구조 확정 | Medium |

> 특히 **1번은 이 결정 없이는 아키 최종 확정이 불가**하다. 별도 실무 회의를 요청드린다.
